3D Gaussian Splatting（3DGS）在静态或弱动态场景里往往重建质量不错，管线也相对清晰。最近在想把问题从「重建真实世界」挪一点：**能不能把昂贵的着色与细节，预装进另一种运行时更省的画法里**——动机上接近引擎里的 lightmap、impostor、LOD 跳变，只是介质换成 **splat**。

<!--more-->

本文是**思想实验**，不是实验报告。我会先区分**传统烘焙**和**用参考渲染做监督的拟合**，用很短一节交代 3DGS 在说什么，再把它和 **MetaHuman**、**LOD** 摆在一起问问题。

## 动机

在偏静态的设置下，3DGS 常常效果很好。工程上老生常谈：**离线多付一点，运行时少付一点**。引擎里用烘焙光照、探针、贴图和 LOD 链一直在做这件事。

这里的问题更窄：**高成本参考**（例如更重的角色渲染）能否给 **更轻的 drawable**（假想为 3DGS，或 GS + 少量 mesh）提供监督，使得在约定相机与条件下，**观感好于单纯廉价几何能给的**？

## 3DGS 原理速览（很短）

只给不熟悉读者的锚点：

- 场景用许多 **三维高斯椭球** 近似：每个有 **位置**，以及描述 **形状与朝向** 的参数（协方差 / 尺度等）。
- 每个高斯还有 **不透明度** 与 **颜色**；颜色常用低阶 **球谐（SH）** 表示，使外观在一段 **视角范围** 内可以变化。
- **渲染**把椭球投到屏幕成 **splat**，按深度排序后做 **alpha 合成**，直觉上接近经典的点 / splat 光栅化思路。
- **训练**从 **多视角图像** 出发，用可微渲染调整各高斯参数，使合成图像贴近观测。

原始表述见 Kerbl 等人的 *3D Gaussian Splatting for Real-Time Radiance Field Rendering*（[项目页](https://repo-sam.inria.fr/fungraph/3d-gaussian-splatting/)）。正文不展开公式，只统一后文用词。

## 这里的「烘焙」指什么

**传统烘焙**多是预计算光照、阴影、GI 等，写进顶点、纹理、探针，运行时光照更省。

本文说的「烘焙」是邻域概念：用 **高成本参考渲染**（或 **更高 LOD** 的资产）产生 **监督**，去 **拟合** 一个 **更便宜** 的渲染对象（3DGS 或混合表征），使其在约定条件下 **贴近参考**。

这不是说 3DGS **等于** lightmap，而是在问：splats 能不能作为一种 **可查询的、预计算外观近似**，落在同一种 **预算交换** 的逻辑里。

## 3DGS 和这种「烘焙」合拍吗？可拆的子问题

下面是讨论点，不是结论。

- **视角与反射**：镜面、皮肤强高光、角膜等，SH 限幅下的外推是否够用；是否需要 **hybrid**（脸仍用 mesh，GS 只管毛发 / 绒毛）。
- **时间 / 动画**：单套「雕像」splat， versus **表情与肢体**变化——若动，是每帧优化、骨骼驱动高斯，还是 **仅在 LOD 切换** 时用 GS 过渡。
- **与 LOD 的关系**：引擎里 LOD 是 **几何与材质切换**。两种叙事容易混：
  - **蒸馏**：**LOD3** 当 **teacher**，把 **接近 LOD0 的观感** 作为 student（例如有上限的 splat）的目标。
  - **外观解耦**：运行时几何仍是 **LOD3**，**观感**由 GS（或由 GS 离线生成的贴图）承担。字面都像「LOD」，合同不一样。

两种都写出来，后文才不会偷换概念。

## MetaHuman 那条具体脑洞

具体画面：

- **Teacher**：**MetaHuman LOD3**（或你定义的「便宜」基准档），在 **固定或窄范围光照**、**固定或窄范围表情** 下渲染并采视角。
- **Student**：primitive 有硬上限——例如 **splat 数封顶**，或 **splat + 低模**。

**假设（不是断言）**：若在限定的 **相机弧** 与 **表情包络** 内能拟合 teacher，是否在 **实时预算** 下得到「读起来像更高档 LOD」的结果；**代价**是存储、视角盲区、动画一致性中的哪些？

**边界**：毛发、半透明、强 SSS 区域可能要 **分阶段** 做，或 **第一轮刻意不做**，避免读成「一键解决 MetaHuman」。

## 风险与反例

- 训练弧外的 **视角外推**；牙齿与口腔；**极近距离**；运动模糊；与引擎 **TAA**、时域滤波的交互。
- 「实时」要落地：**目标机型**、**分辨率**、**毫秒预算**、是否允许 **按视角** 的离线步骤。

## 可以怎么去做（分阶段）

**Phase 0 — 收窄问题：** 仅头部、**静止** 姿态，固定 HDRI 或小棚灯；Teacher 用 UE 里 **LOD3** 出图。交付：稳定 **多视角 RGB**（可选 depth / normal）。

**Phase 1 — 外观能否拟合：** 现成 3DGS（或同类）只对 **静态** 脸；**splat 硬上限**。判据：固定视角弧上指标或主观对比。若失败，先改 **采集密度** 再谈 MetaHuman 叙事。

**Phase 2 — 接上 LOD 故事：** 同一姿态，**LOD0 与 LOD3** 两个 teacher，看 student 更贴谁、差在何处（发际线、高光瓣等）。

**Phase 3 — 最小动态：** 单 **blendshape** 小范围；或粗暴 baseline（**每关键帧一套 splat**）看 **闪断** 与 **存储**。若太差，说明更适合 **hybrid** 而非纯 GS。

**工程路径（可一笔带过）：** 离线训 splat → **引擎可读格式** 回放（若有插件/管线）；或仅在 **离屏** 用 GS 生成 **贴图** 回灌传统材质。笔记里不必先站队。

更有价值的是 **问题图**：当「烘焙」的对象是 **3D 高斯**、它如何与 **LOD 语义** 对齐，以及 **最小实验** 先能证伪什么。这是 **立项笔记**，有数据后再来改口也欢迎。
